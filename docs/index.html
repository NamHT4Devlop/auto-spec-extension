💡 Why Auto Spec Kit?

Traditional AI coding assistants answer one question at a time. Auto Spec Kit runs a complete, multi-step workflow for each task — from requirement to production-ready deliverable in a single command.

Without Auto Spec KitWith Auto Spec Kit
Manually prompt Copilot for a plan13-step pipeline runs automatically
No code reviewTwo-phase review: quality + business consistency
No tests writtenTest scaffolding generated for every task
No documentationEvidence files saved with every session
AI has no project contextKnowledge Base grows with every task
PO has to write stories manuallyEpic → Sprint Board HTML generated in minutes

📦 Requirements

  • ✓ VS Code ^1.90.0 — any edition (stable, insiders)
  • ✓ GitHub Copilot subscription — Individual or Business/Enterprise
  • ✓ GitHub Copilot Chat extension — installed and signed in
  • ✓ Node.js — only if building from source (not needed for VSIX install)

Uses the vscode.lm API to call Copilot directly inside VS Code — no Anthropic, OpenAI, or other API keys required.

🔧 Installation

Option A — From VSIX (recommended for most users)

  1. Go to Releases page and download auto-spec-kit-1.6.0.vsix
  2. Open VS Code → Extensions panel (Ctrl+Shift+X / Cmd+Shift+X)
  3. Click the ⋯ (three dots) button at the top right of the Extensions panel
  4. Select "Install from VSIX…"
  5. Browse to and select the downloaded .vsix file
  6. Reload VS Code when prompted — the extension is now active

Option B — From Source (for contributors)

# 1. Clone the repository
git clone https://github.com/NamHT4Devlop/auto-spec-extension.git
cd auto-spec-extension

# 2. Install dependencies and compile TypeScript
npm install
npm run compile

# 3. Package into a .vsix file
npx @vscode/vsce package --no-dependencies

# 4. Install the generated .vsix from the Extensions panel (Option A step 3–6)

⚡ Quick Start

  1. Install the extension using Option A or B above
  2. Open a project in VS Code — any language or framework
  3. Press Ctrl+Shift+B / Cmd+Shift+B → "Generate Knowledge Base"
    Auto Spec Kit scans your entire project and creates knowledge-base/ with 16 analysis files. Takes ~3–10 minutes depending on project size.
  4. Press Ctrl+Shift+K / Cmd+Shift+K → "Run Task"
    Type your requirement, e.g. "Add reset password with email OTP, expires after 10 minutes"
  5. Watch the 13-step pipeline run in the Output panel. When finished, review results in spec-kit-sessions/<timestamp>/

💡 Tip: Generate the Knowledge Base first — the pipeline is significantly more accurate when it knows your actual codebase architecture, patterns and business rules.

📖 Detailed Workflows

1. 🚀 Run Task — 13-Step Pipeline 💻 Developer

The core command. Give it one task description and it runs 13 sequential AI steps:

#StepWhat it does
01AnalysisUnderstands the requirement; checks if KB/files exist
02Task PlanWrites a detailed implementation plan
03Plan ReviewReviews the plan for correctness and completeness
04Code GenerationGenerates all code changes / new files
05Code ReviewTwo-phase: quality checklist + business consistency
06Fix IssuesApplies fixes found in the code review
07TestsWrites unit tests and integration tests
08Test ReviewReviews test quality and coverage
09Fix TestsApplies test improvements
10Run TestsExecutes autoSpecKit.testCommand and captures output
11Fix Failing TestsDiagnoses and fixes any test failures
12EvidenceBuilds final evidence document (plan + code + tests + results)
13Update KBMerges new learnings back into the Knowledge Base

After Step 04 (Code Generation) you will be asked: Apply to project now (files written to workspace) or Save in session folder only.

2. 📚 Generate Knowledge Base 💻 Developer

Scans your entire project and generates 16 Markdown files covering every aspect of the codebase at business depth:

FileContent

02-tech-stack.mdLibraries, frameworks, why each was chosen
03-project-structure.mdFolder layout, module boundaries, layer rules
04-domain-model.mdCore entities, relationships, domain concepts
05-api-contracts.mdREST/GraphQL endpoints, request/response shapes
06-auth-security.mdAuth flows, guards, permission model
07-data-layer.mdDB schema, migrations, ORM patterns
08-integrations.mdExternal services, queues, webhooks
09-config-envvars.mdEnvironment variables, feature flags, constants
10-core-flows.mdKey business workflows end-to-end
11-error-handling.mdError types, HTTP codes, error propagation
12-testing-strategy.mdTest coverage, patterns, test data setup
13-business-rules.md ★Business invariants, validations, constraints
14-performance-scalability.mdBottlenecks, caching, scaling patterns
15-tech-debt.mdKnown issues, TODOs, areas needing attention
review-skills.md ★Project-specific rules merged with universal template

3. 🔍 Review Current File 💻 Developer

Right-click any open file → Auto Spec Kit: Review Current File. Runs a two-phase deep review:

Phase 1 — Code Quality: Security · Architecture · Performance · Error Handling · Code Quality · Type Safety · Testability · API Design · Project-specific rules

Phase 2 — Business Consistency: Checks against 13-business-rules.md — are business invariants intact? Any logic silently deleted? State transitions valid?

Output includes: section coverage table, issues with before/after code, strengths list, verdict (APPROVED / NEEDS_REVISION), quality score /10.

4. 📋 Generate User Stories (PO/BA) 📋 PO / BA

Press Ctrl+Shift+U. A 4-step pipeline for Product Owners and Business Analysts:

StepOutput
InvestigationDomain analysis: actors, business rules, edge cases, technical constraints
User StoriesFull story list with roles, actions, benefits, story points, priorities
Acceptance CriteriaGiven/When/Then ACs + Definition of Done for each story
Sprint PlanningStories grouped by sprint, sequenced by dependency and risk

Saves a JSON file and an interactive HTML sprint board to spec-kit-sessions/ — with priority filters, role filters, expandable story cards, and PDF export.

⚙️ Configuration Reference

Open Ctrl+, → search "Auto Spec Kit", or edit settings.json directly:

{
  // Command to run your test suite.
  // Output is captured and analyzed by Copilot in Step 10 (Run Tests).
  // Leave blank to skip the test-run step.
  // Examples: "npx jest --coverage 2>&1" · "npm test 2>&1" · "python -m pytest 2>&1"
  "autoSpecKit.testCommand": "",

  // Primary programming language — guides code style and test framework hints.
  // Options: "typescript" | "javascript" | "python" | "java" | "go"
  "autoSpecKit.language": "typescript",

  // Relative path to the Knowledge Base folder (from workspace root).
  "autoSpecKit.knowledgeBasePath": "knowledge-base",

  // When true, generated code is written to the project without the apply/skip prompt.
  "autoSpecKit.autoApplyCode": false,

  // Folder where session outputs are saved (relative to workspace root).
  "autoSpecKit.sessionsDir": "spec-kit-sessions",

  // Preferred Copilot model ID. Leave blank for auto-selection (best available).
  // Run "Auto Spec Kit: Select Model" to browse all options.
  // Examples: "gpt-5.5" · "gpt-5.4" · "claude-opus-4-7" · "gemini-2.5-pro" · "o3"
  "autoSpecKit.model": "",

  // When true, shows model Quick Pick at the start of every command run.
  "autoSpecKit.askModelOnStart": false
}

Recommended setup for most projects:

{
  "autoSpecKit.language": "typescript",
  "autoSpecKit.testCommand": "npx jest --coverage --coverageReporters=text 2>&1",
  "autoSpecKit.model": "gpt-5.5",
  "autoSpecKit.autoApplyCode": false
}

🧠 Knowledge Base

The Knowledge Base is the foundation of Auto Spec Kit — a folder of Markdown files (knowledge-base/) that:

  • ① Powers code generation — Copilot knows the exact architecture, patterns, and business rules before writing new code
  • ② Powers code review — every review cross-references business rules and domain constraints
  • ③ Powers Q&A — the Ask command answers questions grounded in actual codebase evidence
  • ④ Powers planning — task plans reference existing flows and avoid duplicate implementations
  • ⑤ Gets smarter over time — every completed task updates the KB automatically at Step 13

💡 Tips & Best Practices

Getting the best results from Run Task:

  • → Be specific. Instead of "add auth" write "Add JWT email/password auth with refresh tokens, bcrypt hashing, and rate limiting on login."
  • → Generate KB first. The pipeline is significantly more accurate when it references your actual codebase architecture.
  • → Set autoSpecKit.testCommand so Step 10 runs your real tests and Step 11 can fix actual failures.
  • → Read Step 05 output carefully. The two-phase review often surfaces issues a manual review would miss.

Knowledge Base tips:

  • → Re-run Generate KB after major refactors or architecture changes.
  • → The most valuable KB file is 13-business-rules.md — the more accurate it is, the better every review becomes.
  • → Commit review-skills.md to share your project's review standards with the entire team.

User Stories tips (PO/BA):

  • → Provide a detailed Epic Description including non-functional requirements and explicit out-of-scope items.
  • → 5–10 features per epic produces the most useful story granularity.
  • → The generated HTML sprint board can be printed as a PDF for stakeholder review sessions.
