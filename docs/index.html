🧭 Clarity & Ambiguity Handling — v1.8.4

Gets the best result even when your prompt is vague — by clarifying before doing expensive work instead of guessing.

FlowBehavior on vague input
Build (chat)Scores requirement clarity; if below clarify.threshold it asks 2–3 targeted questions and halts — no wasted 13-step build. Refine and re-run.
Build (Command Palette)Now also clarifies: prompts via input boxes and merges your answers into an enriched requirement before building.
Ask / investigateDetects vague questions; the answer states its interpretation + assumptions, answers the most likely intent, then offers focused clarifying questions — and never fabricates beyond the KB. No extra model call.
Configclarify.enabled (on/off) and clarify.threshold (0–100 strictness).

⚡ Token Optimization — v1.8.3

Reduces token usage for everyday tasks without hurting output quality. All tunable in settings.

AreaChange
Ask / investigateEach question loads only the relevant KB topics (capped by ask.maxContextTokens) instead of the entire knowledge base — typically 60–85% fewer tokens per question, no extra model call.
Build system promptKB embedded in the system prompt (reused every step) is sized by contextStrategy: smart caps it (systemKbTokens), minimal omits it, full keeps all.
Sub-agent mergemergeStrategy: auto (new default): review steps merge deterministically (no extra model call); generative steps keep AI merge — removes ~4 model calls per build.
Settings wiredcontextStrategy, mergeStrategy, maxParallel now actually applied to the build pipeline (previously declared but unused).

What is Auto Spec Kit?

A VS Code extension that turns a one-line task description into a complete, production-ready deliverable. It orchestrates GitHub Copilot through a structured multi-agent pipeline — from planning to code generation, code review, test writing, and evidence collection — while keeping a growing Knowledge Base that makes every subsequent task smarter.

Multi-Agent Architecture (v1.7)

Each pipeline step spawns parallel sub-agents that analyze different aspects simultaneously, then a merge agent synthesizes the results. This produces deeper, more accurate output than a single AI call.

Deep Dive: @autospec /build

When you type @autospec /build Add reset password feature using email OTP, expires after 10 minutes, this is the full execution path:

How the Chat Participant bridges to the pipeline

  1. You type @autospec /build Add reset password feature in Copilot Chat
  2. VS Code routes the message to the @autospec Chat Participant (registered in chat-participant.ts)
  3. GitSyncGuard runs git fetch --all --prune + git pull --ff-only. If source files changed, the KB is auto-updated. Non-fatal — skipped silently if offline or not a git repo.
  4. handleBuild() calls resolveModel() to pick the best available Copilot model, then creates a Progress adapter that converts vscode.Progress.report() calls into stream.progress() — so every pipeline step updates the chat panel in real-time
  5. handleBuild() calls runWorkflow() from workflow/run-task.ts — the same function used by the Command Palette command
  6. runWorkflow() creates a PipelineRunner and executes ALL_STEPS (13 step classes). Each step spawns sub-agents via AgentOrchestrator with SmartContextLoader-selected context
  7. 3 human checkpoints (steps 03, 06, 09) pause the pipeline with VS Code Quick Pick dialogs — you approve, add comments, or cancel
  8. When the pipeline completes, handleBuild() writes a final completion message to the chat stream

🧑‍💻 Developer Flow — Build Feature

FOR: Developers

The core command. Give it a task description and it runs a 13-step multi-agent pipeline from planning to evidence collection. Press Ctrl+Shift+K to start.

Key Features

  • Checkpoint Resume — If pipeline fails at step 7, you can resume from step 8 next time. State saved to .pipeline-state.json.
  • Smart Context — File Discovery Agent identifies relevant files before each step. Each sub-agent gets only the context it needs.
  • Token Budget — Auto-manages context size to prevent model overflow.
  • Transparency — Individual agent outputs saved as agent-*.md in session folder for debugging.

🔍 Code Review Flow

FOR: Developers

Right-click any file → Auto Spec Kit: Review Current File

  1. Load file content + git diff context (branch changes + working tree)
  2. Load review skills (KB version or universal template)
  3. 4 parallel review agents:
    🔒 Security Reviewer — injection, auth bypass, data exposure
    🏗 Architecture Reviewer — patterns, coupling, layer violations
    ⚡ Performance Reviewer — N+1 queries, memory leaks, bottlenecks
    🏢 Business Reviewer — rules intact, state machines valid, logic preserved
  4. Merge Agent produces final review: Section Coverage table + Business Consistency table + Issues with fixes + Verdict + Score
  5. Review opens in a new panel beside the editor

📊 SM / Tech Lead Flow — Knowledge Base & Graph

FOR: Scrum Masters, Tech Leads, Architects

Scan Project (Ctrl+Shift+B)

Scans the entire project and generates 16 analysis files with multi-agent deep analysis on critical business files.

Source-Only KB Scan Mode

When existing documentation (README, docs/, .github/, copilot-instructions.md) is outdated or misleading, you can rebuild the Knowledge Base purely from source code.

On scan, a QuickPick prompt offers two options:

  • "Fresh rebuild — source code only" — excludes all existing docs, generates KB from source files only
  • "Regenerate KB (include existing docs)" — default behavior, includes documentation in analysis

Configure in .autospec.yml:

scan:
  excludeDocs: true          # skip README, docs/, .github/, copilot-instructions.md
  exclude:
    - "generated"            # additional folders to exclude
    - "vendor"

🔭 Map Codebase

FOR: Architects, Tech Leads

🔄 GitSyncGuard — Auto-Sync Before Every Command

Before every command (except /help), GitSyncGuard automatically fetches and fast-forward merges the latest changes from the remote. If source files changed, the Knowledge Base is updated automatically.

What it doesWhat it NEVER does
git fetch --all --pruneNever pushes to remote
git pull --ff-only (safe fast-forward)Never commits changes
Detects changed source files after pullNever writes to git remote
Auto-updates KB when source changedNever force-merges or rebases
Works silently — non-fatal on errorsNever blocks a command on failure

Configuration

// VS Code Settings
"autoSpecKit.autoSync": true,         // enable/disable auto-fetch/pull
"autoSpecKit.autoSyncKBUpdate": true   // auto-update KB after sync

// .autospec.yml
autoSync:
  enabled: true
  kbUpdate: true

Offline-safe: If git fetch fails (no network, no remote, not a git repo), the command proceeds normally. GitSyncGuard never blocks your workflow.
